% Although Reinforcement Learning provides the basis for the research in this paper, it is generally applied to tasks that are sequential in nature. For tasks where the transition to the next state can be ignored, bandits are a much better framework to represent the problem. There are several types of bandit representations, and for tasks with multiple actions, the resulting framework is the multi-armed bandit. The contextual bandit extends this to situations with multiple states or contexts, and is the chosen framework in this paper.

\section{Contextual Bandit Framework}
\label{Framework}
The method in this paper was designed to be applied to a one-step, no horizon task, which is described in the Experiments \ref{Experiments} section. Because of this, we opt to utilize the contextual bandit \cite{Bandit} framework rather than the traditional Markov Decision Process \cite{MDP} framework. The two frameworks are very similar, but in the contextual bandit setting, there are no state transitions. Because of this, there is no transition function and only the immediate expected reward is considered rather than the sum of expected rewards.

The contextual bandit framework is formally defined by the following: there are a number of rounds $t$ from $t=1, 2, 3, ..., T$. For the set of possible contexts $\mathcal{X}$ there is a context vector $x_t \in \mathcal{X}$ to describe the current trial's context or state. There is also a set of arms or actions $\mathcal{A}(x_t)$, and the agent must choose one action $a_t \in \mathcal{A}(x_t)$ during each round. After executing the action, a reward $r_t$ is observed and determined by the reward function $r(x,a)$. In order to choose actions for each individual context, the agent implements a policy $\pi(a|x)$ or mapping from context to action. With a deterministic policy, each context results in a single action $a=\pi(x)$ with a probability of 1. For a deterministic policy represented by parameters $\phi$, the goal is to find a policy that maximizes the objective or expected reward

\begin{equation}
J(\phi) = 
\mathbb{E}_{x \sim p_x(\cdot)}
\big[r \big( x, \pi_\phi (x) \big) \big].
\label{J}
\end{equation}

By repeatedly taking the gradient of the objective $\nabla_\phi J(\phi)$, the policy parameters can be continuously updated and push the policy towards optimality. The deterministic policy gradient from \cite{DPG} can then be defined by the following

\begin{equation}
\nabla_\phi J(\phi) = 
\mathbb{E}_{x \sim p_x(\cdot)}
\big[\nabla_\phi \pi_\phi(x) \cdot \nabla_a r(x, a)|_{a=\pi_\phi(x)} \big].
\label{GJ}
\end{equation}

Please note that the original deterministic policy gradient is applied to actor-critic RL algorithms, so in the above, the the Q-function $Q^\pi(s,a)$ has been replaced with the reward function. The Q-function represents the value for taking action $a$ in state $s$ and following the policy $\pi$ thereafter. The thing that makes the Q-function different from the reward function is that the Q-function captures the total discounted sum of future rewards while the reward function represents only the immediate reward. Because this paper deals with the formulation of contextual bandits rather than MDP's, there are no transition and resulting states to worry about, so the replacement of the Q-function with the reward function in Equations \eqref{J} and \eqref{GJ} is justified. 

The deterministic policy gradient can be estimated from executing the policy $\pi_\phi$ in the real-world and adding the observed data to a dataset $\mathcal{D}^{\pi_\phi}=\{(x_i,a_i,r_i)\}_i^{\|\mathcal{D}^{\pi_\phi}\|}$. The dataset $\mathcal{D}^{\pi_\phi}$ can be used to estimate the deterministic policy gradient with the following

\begin{equation}
\nabla_\phi J(\phi) =  
\frac{1}{\|\mathcal{D}^{\pi_\phi}\|} \sum_{b=1}^{\|\mathcal{D}^{\pi_\phi}\|}
\nabla_\phi \pi_\phi(x_b) \cdot 
\nabla_ar(x_b,a_b) |_{a_b=\pi_\phi(x_i)}.

\section{Reward Function Modeling}
\label{RFM}

As mentioned above, the observed reward $r_t$ obtained after taking action $a_t$ in context $x_t$ is determined by the reward function $r(x,a)$. In many practical settings, however,  the true reward function is unknown to the agent and must be inferred through interactions with the environment. One approach is to directly learn a policy from experience by utilizing importance sampling, but an alternative is to learn an approximate reward function $\hat r(x,a)$ and use this for policy optimization. For any exploration policy $\pi_{\text{Explore}}$, the agent observes tuples of the form $(x_t, a_t, r_t)$ where $x_t$ is the context, $a_t$ is the policy action $a_t\sim\pi_{\text{Explore}}(\cdot|x_t)$, and $r_t$ is the resulting reward. These interactions can be used to generate a dataset $\mathcal{D}=\{(x_b,a_b,r_b)\}_{b=1}^{\|\mathcal{D}\|}$. After populating the dataset, modeling the approximate reward function becomes a supervised learning problem. 

For an approximate reward function $\hat{r_\theta}(x,a)$ with parameters $\theta$, the goal is predict the true reward of each context-action pair. To train this model, the aim is to minimize a loss function $\mathcal{L}(y,\hat y)$ which quantifies the discrepancy between true labels $y$ of input $x$ and predicted labels $\hat y$ from a function $f_\theta(x)$ \cite{19}. In the reward model setting, the loss function would be defined as $\mathcal{L}(r_b,\hat{r}_\theta(x_b,a_b))$, which measures how well the predicted reward matches the true observed reward. The optimal parameters $\theta$ are obtained by minimizing the loss over a mini-batch of size $B$ sampled from the dataset $\mathcal{D}$: 

\begin{equation}
\theta = 
\arg\min_\theta
\frac{1}{B} \sum_{b=1}^B
\mathcal{L}(r_b,\hat{r}_\theta(x_b,a_b)).
\label{Loss}
\end{equation}

This gradient-based optimization updates the parameters $\theta$ so that the learned reward function $\hat{r}_\theta(x,a)$ accurately approximates the true reward. Once trained, the approximate reward function can be used for policy optimization, enabling sample-efficient learning without the true reward function.
